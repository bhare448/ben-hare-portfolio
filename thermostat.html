Smart Thermostat Wireframe

Wireframe for a smart thermostat made for my Interaction Design Studio class.

Project Overview

This project was assigned in my "Interaction Design Studio" class and was meant to teach us how to effectively demonstrates changes in state when it comes to digital interfaces. Our goal was to design a thermostat for a smart home with at least 3 different options and use different signifers to communicate the change in state to the user.

Design Process

Ideation:

  • Since there are a large number of smart thermostats on the market, the ideation process for this project was fairly simple, as I just needed to include at least 3 different states that many thermostats already have built in and make sure that is still functions correctly as a thermostat.
  • The difficulty came with deciding which kinds of signals I wanted to include in the design. I wanted to make sure that users were able to perceive changes in state from a long distance since they would likely not want to walk over to their thermostat just to see what mode it is in.

Research:

  • Research was an important step in this project because I was unfamiliar with how thermostats worked and wanted to make sure that I included the correct functions on it.
  • I also looked into different ways to save money with heating since our professor wanted us to make that a priority in our design. My research indicated that the best way to save money is to use less power on their thermostat when the user is out of the house or sleeping.

Brainstorming:

  • After some consideration, I figured that a combination of symbols and color would be most effective in signifying state since different colors invoke different emotions in people and can be easily understood by a wide range of audiences.
  • The 3 modes that I decided to include were "Indoor Mode", "Outdoor Mode" and "Sleep Mode". "Indoor Mode" shows the indoor temperature and can be changed at any point. "Outdoor Mode" displays the current temperature outside to let the user know what they should set their internal temperature to. Lastly, "Sleep Mode" allows the user to set the temperature to go up or down at a certain time, typically when they are asleep, in order to save money.

Prototype:

  • The wireframe for this product consists of an LED screen that functions as the main display, a power button on the right side, arrow buttons on the left side to increase or decrease the temperature, 3 buttons on the bottom with icons to change between the 3 different modes, and a tip button to provide tips for each of the modes.
  • Color is used as the primary signifier with a neutral gray color for "Indoor Mode", a light blue color for "Outdoor Mode" and a dark teal color for "Sleep Mode".
  • The main purpose of this device is to help users save money on their heating bill by recommending temperatures to set their house to based on the current temperature outside.

Final Outcome

The final version of this wireframe turned out well and included all of the criteria for the project. This project taught me the importance of signifiers and changes in state, which I continue to use in many of the designs I make today. It helped develop the skills needed for empathizing with users and designing in a way that will benefit them the most.
